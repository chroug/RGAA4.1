Erreur 2 : Perte de sens à la linéarisation (Le cauchemar des lecteurs d'écran)

Le développeur a coupé une phrase en deux pour faire deux colonnes visuelles. Visuellement, on lit la colonne 1, puis la colonne 2. Mais le code HTML lit Ligne 1 (Col A + Col B), puis Ligne 2. La phrase lue sera absurde.

Erreur 3 : Ordre d'un formulaire brisé

Le développeur a mis tous les labels dans la première ligne, et tous les champs dans la deuxième ligne. Le lecteur d'écran va lire "Nom", puis "Prénom", et ENSUITE il arrivera sur les champs sans savoir lequel est lequel.
